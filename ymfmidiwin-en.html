Software OPL FM MIDI synthesizer
ymfmidi for Windows

Download ymfmidi for Windows

ymfmidi for Windows is a software FM MIDI synthesizer for Windows, based on ymfm and ymfmidi.

Unlike FM-style soundfonts, this software performs true FM sound synthesis by emulating YAMAHA OPL-series FM chips at the waveform synthesis level.

What is “FM MIDI”?

In the past, many PCs used FM sound chips to play MIDI music. These chips generated sound by combining simple waveforms, resulting in a distinctive and hardware-dependent sound character.

Modern MIDI synthesizers usually rely on sampled instruments, but FM synthesis still has unique qualities that cannot be reproduced exactly with soundfonts.

This project aims to reproduce the original FM synthesis behavior itself, not just imitate its sound.

Features

  • True FM sound synthesis
    • Emulates YAMAHA OPL / OPL2 / OPL3 chips
    • Waveform-level FM synthesis (not sample-based)
    • Multiple virtual chips can be used to increase polyphony
  • Improved MIDI compatibility
    • Supports GM, GS, and XG behavior (depending on instrument definitions)
    • Correct handling of MIDI specifications that were not properly supported by older Windows FM MIDI drivers (e.g. pitch bend range)
  • Native Windows MIDI device
    • Provides a resident software synthesizer usable as a MIDI IN device
    • Can be used from standard Windows applications supporting the midiOut API by using a virtual MIDI cable such as loopMIDI
  • Real-time playback and offline rendering
    • Real-time audio output via WASAPI
    • MIDI-to-WAV rendering supported

Information for those who want Windows-style FM MIDI sound

If you want to play MIDI with tones similar to the standard FM MIDI sound source in Windows, you will need the tone parameters used by the standard Windows Sound Blaster 16 driver. The actual data is included as FMSYNTH.BIN in the Windows NT4 DDK sample.

However, since this parameter definition was originally provided for driver development (DDK) use, redistributing it for purposes outside that scope may be inappropriate—even if it is only a sample. For this reason, this software does not include FMSYNTH.BIN. Instead, the following alternative methods are supported:

  1. Using the FMSYNTH.BIN from the DDK sample

    If you specify the FMSYNTH.BIN included with the Sound Blaster 16 sample driver in the Windows NT4 DDK as the tone definition file, it can be loaded.

    Example: ymfmidiwin //MIDIIN FMSYNTH.BIN

  2. Loading from a driver DLL

    In fact, the Sound Blaster–series driver DLLs for Windows NT4 (such as sndblst.dll) contain the same data as FMSYNTH.BIN embedded as a resource.

    Therefore, this software can also load tone data from such DLL files if they contain an FMSYNTH.BIN-equivalent resource. It does not have to be specifically sndblst.dll, as long as the resource is present.

    Example: ymfmidiwin //MIDIIN sndblst.dll

[Note] The drum tones contained in FMSYNTH.BIN depend on OPL rhythm sounds. However, rhythm sounds are currently not supported in ymfmidi. For this reason, only the drum tones are loaded from GENMIDI.op2 instead. Please place GENMIDI.op2 in the same folder as ymfmidiwin.exe.
